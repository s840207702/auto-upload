# 一键整合包底座选型说明

## 当前选择

当前一键包先采用 `zydgmail/social-auto-upload` 作为 MVP 底座。

原因：

- 已经包含构建好的 Vue 前端 `frontend/dist`。
- 后端是单入口 Flask `main.py`，更容易打成一键包。
- 已经有账号管理、素材管理、发布中心、状态验证等基础闭环。
- 目录结构简单，适合先做“能启动、能登录、能上传、能发布”的新手版本。

## 不选当前 Qt 壳的原因

从零做桌面 GUI 会反复遇到底层复杂问题：

- 浏览器生命周期管理。
- 账号登录态保存和刷新。
- 二维码登录状态回传。
- Cookie 失效检测。
- 发布任务队列。
- 失败重试和日志追踪。

这些不是简单 UI 能解决的问题，应该基于已有 Web 管理系统继续二开。

## MediaPublishPlatform 的价值

`funfan0517/MediaPublishPlatform` 更适合作为长期架构参考。

它值得吸收的部分：

- `platform_configs.py` 平台能力配置。
- 发布任务记录表。
- 统一登录和统一验证。
- Cookie 上传/下载。
- 多文件、多平台、多账号发布接口。
- 发布失败后的记录、取消、删除、重试思路。

## 当前 MVP 目标

先做一个适合新手的本地一键包：

- 双击启动。
- 自动检查依赖。
- 自动打开浏览器页面。
- 能添加账号。
- 能上传素材。
- 能进入发布中心。
- 能保存本地数据。

后续再逐步增强：

- 环境检查页。
- 登录失败诊断。
- 发布失败重试。
- 平台 selector 配置化。
- 更清爽的新手 UI。
- 打包成 zip 或 exe 分发包。
