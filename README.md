## 项目定位

`非丨自媒体一键分发` 不是云端代运营系统，而是一个更偏本地工作台的创作者工具。它把多个平台分散、重复、容易出错的发布准备动作收敛到一个 Web 管理界面里，让创作者可以更稳定地完成素材上传、封面配置、文案填充、话题处理、定时设置和发布前检查。

项目默认优先保护本地数据：账号登录态、素材、封面、日志和数据库都保存在用户自己的运行环境中，不随源码仓库提交。

> 本项目仅供技术研究、学习交流和个人内容管理效率提升使用。请遵守各平台用户协议、社区规范、内容发布规则及所在地法律法规。

## 能力概览

| 模块 | 能力 |
| --- | --- |
| 发布中心 | 多平台选择、账号主体、素材队列、平台封面、公共文案、话题、立即/定时发布 |
| 预发布检查 | 调试阶段只执行上传、填表、封面和定时设置，停在最终发布前，便于人工验收 |
| 批量流程 | 按 `抖音 -> 视频号 -> B站 -> 小红书 -> 快手` 的顺序执行多平台发布准备 |
| 定时发布 | 支持开始天数、每天条数、基准时间和随机浮动分钟 |
| 封面适配 | 针对平台差异处理横封面、竖封面、双封面和平台当前比例 |
| 账号管理 | 支持本地登录态保存、账号状态刷新、多平台后台标签页打开 |
| 失败收口 | 单个平台某条视频失败后停止该平台后续视频，避免半完成任务继续推进 |

## 支持平台

| 平台 | 当前适配能力 |
| --- | --- |
| 抖音 | 视频上传、作品简介、话题、横/竖封面、定时发布、预发布检查 |
| 视频号 | 视频上传、描述与话题、短标题、个人主页卡片、分享卡片、定时发布、预发布检查 |
| B站 | 视频上传、稿件标题、简介、标签、分区、创作声明、双封面、定时发布、预发布检查 |
| 小红书 | 视频上传、作品简介、话题、封面比例选择、定时发布、预发布检查 |
| 快手 | 视频上传、作品描述、话题、封面比例选择、定时发布、预发布检查 |

仓库中仍保留部分原项目平台目录，实际可用能力以当前发布中心和各上传器适配状态为准。

## 相比基础版本的主要增强

### 发布体验

- 重构发布中心信息架构：账号主体、平台选择、素材、封面、作品文案、话题和发布方式分区更清晰。
- 将最终发布动作放回流程底部，更符合用户完成检查后再提交的操作逻辑。
- 发布素材改为稳定列表形态，避免文件数量变化导致组件高度大幅抖动。
- 平台封面区域重新梳理为固定展示空间，减少不同尺寸、不同数量封面造成的布局跳变。
- 公开“作品文案”改为可选；B站单独提供必填“稿件标题”和独立简介，更适合多平台差异化发布。

### 平台适配

- 抖音：横封面 `4:3`、竖封面 `3:4` 分别处理。
- 视频号：个人主页卡片 `3:4`、分享卡片 `4:3` 分别处理。
- B站：首页推荐 `4:3`、个人空间 `16:9` 双封面处理，并适配新版创作声明、分区、标签和简介流程。
- 小红书、快手：按平台当前选择比例上传对应封面，并针对话题节点生成做等待、回读校验和重试。

### 稳定性

- 自动化浏览器窗口最大化显示，减少固定尺寸导致页面边缘被截断的问题。
- 增加预发布检查模式，调试时只停在最终发布前，不直接点击平台发布按钮。
- 部分平台增加定时写入后的回读校验。
- 多条视频批量处理时增加失败收口，避免异常后继续推进导致状态不可控。

国内平台发布页变化较快，选择器和交互流程需要持续维护。本项目会优先保证当前主流发布页机制的稳定性。

## 快速开始

### 环境要求

- Windows 10/11
- Python 3.10+
- Node.js 18+，仅前端开发时需要
- Chrome / Edge / Playwright Chromium

### 启动

```bat
start-oneclick.bat
```

或：

```bat
run.bat
```

启动后访问：

```text
http://127.0.0.1:5409/
```

## 配置说明

公开仓库提供 `conf.example.py` 作为配置参考。首次运行前可以按需复制为 `conf.py` 并修改本地浏览器路径、调试开关等参数。

```python
LOCAL_CHROME_PATH = ""
USE_SYSTEM_BROWSER = False
DEBUG_SKIP_FINAL_PUBLISH = True
```

调试阶段建议保持：

```python
DEBUG_SKIP_FINAL_PUBLISH = True
```

这样系统只完成上传、填表、封面和定时设置，不点击各平台最终发布按钮，便于人工检查。

## 本地数据与隐私

以下目录属于运行时数据，不应提交到公开仓库，也不建议打包进源码发布物：

| 路径 | 内容 |
| --- | --- |
| `cookiesFile/` | 平台登录态和 Cookie |
| `db/` | 本地数据库 |
| `videoFile/` | 上传视频、封面和临时处理文件 |
| `avatars/` | 账号头像缓存 |
| `logs/` | 运行日志和调试截图 |
| `frontend/dist/` | 前端构建产物 |
| `.venv/` | 本地虚拟环境 |
| `python/` | 一键包内置 Python 环境 |
| `third_party/` | 浏览器、ffmpeg 等第三方运行时依赖 |

本项目不保存、不收集用户账号密码。登录态、Cookie、素材文件和发布记录默认仅存储在用户本地环境中。
